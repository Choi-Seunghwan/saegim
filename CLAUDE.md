# 새김 (SAEGIM) — 프로젝트 지침

- 대화는 **한글**로 할 것
- 주석은 **한글**로 달 것
- 새김(SAEGIM)은 **문장을 카드로 만들어 발견·새김·컬렉션으로 엮는** 모바일 웹 서비스. 현재 **기획/디자인 단계**(개발 착수 전)인 독립 프로젝트.
- 작업 시작 전 `PLANNING.md`를 읽어 최신 기획·결정 맥락을 파악할 것 (단일 진실 소스).
- 프로젝트 문서 허브는 `docs/README.md`다. 개발·환경변수·인프라·배포 상태는 여기서 필요한 문서로 들어간다.

## 구조
- `PLANNING.md` — 기획 노트(결정/진행 로그). SSOT.
- `design-system.html` — 기획·디자인 시스템 + 화면별 데모(app.html 딥링크 임베드).
- `saegim/app.html` — 통합 앱(메인 데모, 단일 파일 SPA, 바닐라 JS).
- `saegim/editor.html` — 카드 에디터(작성=조회 WYSIWYG).
- ※ git 관리 시작(정리 전 기준선 커밋 존재). 초기 탐색물(concepts/·mockups/·gallery·mvp-mockups·단독 화면 데모·브랜드 보드)은 정리로 제거 — 필요 시 git 이력에서 복원.

## 정체성 (요약)
- **하이브리드 톤**: 새김의 감성('새김'·'새기다'=마음에 새겨 간직)은 유지하되, 비주얼·공유는 트렌디(인스타 감성카드)하게. **판각/도장(刻) 모티프는 제거**(새김 발행 모션만 잔존). 'C 방향' 확정.
- **비주얼 = C(소프트 감성) · 모노톤 포인트**: 뉴트럴 미스트(#F6F5F6) 배경 · 먹보라/차콜 잉크(#38323F) · **포인트=잉크 차콜(#353039) 모노톤**(분홍 제거 — 색은 '글 카드'에만, UI 크롬은 무채색). 카드 배경은 **새벽/노을/안개/살구/라벤더/밤** 그라데이션 프리셋(에디터 기본=안개). 폰트 = 카드 문장 **Gowun Batang**(소프트 명조) · 워드마크 **Gowun Dodum**(잉크 밑줄) · UI **Pretendard**. ※ 첫 적용 시 전반이 너무 분홍→모노톤으로 정정(2026-06-27).

## 핵심 모델 (요약)
- **카드(장)** = 원자 단위(한 종류, **카드 내부 스크롤 없음** — 작성 크기로 표시·넘치면 클립). 발행 단위(게시물)의 UI 명칭 = **글** = 카드(장)의 순서 묶음(1~N장).
- **카드(장) ↔ 글 = 1:N**, 발행 단위는 늘 **글**(1장도, N장도 한 '글'). 'UI 용어=글/장', 내부 코드 식별자(COLLECTIONS 등)는 유지.
- **결(테마/무드)은 사용자가 직접 안 고름**(카테고리 강요 지양). 비슷한 문장 묶기·추천·취향은 **의미 임베딩(백엔드)** 이 담당(프로토타입은 클라 similar 대체). 보조 **태그**(사용자 자기 말, 0~3, 선택)만 발행 시 입력.
- 발견 = 글 피드(세로=글, 가로=장). **둘러보기**(구 '서가') = 글 갤러리. 프로필 지표 = **글 수 + 글벗**(구독자→'글벗'으로 명칭 순화, 경쟁 톤↓). **문장 수는 비노출**(글과 혼동). 구독 버튼 라벨='구독/구독중'(곁에 두기 문구 미사용).
- **좋아요 ↔ 새김(서랍) 분리**(둘 다 유지): **좋아요(♡)=공개 공감**(남에게 보내는 가벼운 반응·수치 공개), **새김(서랍에 넣기)=비공개 간직**(나만 보는 보관함=내 서랍에 담아 다시 봄, 수치 비공개). 한 줄로 "보내는 것(좋아요) vs 담는 것(새김)". 새김이 브랜드 핵심 액션.
- 5탭(홈·발견·＋포착·둘러보기·나, 라벨 없는 아이콘만·나=프로필 사진) + 검색. 소셜은 **인스타식 평등 계정**(인증 ✓는 상태). 프로필은 나=타인 동일 화면(나만 우측 상단 옵션).

## 작업 규칙
- 데모는 의존성 없는 **단일 HTML**(바닐라 JS, **C 소프트 감성** 디자인 토큰). CDN 폰트 = Pretendard·Gowun Batang·Gowun Dodum·Nanum Pen Script·Black Han Sans(카드 문장 5종: 고딕/명조/둥근/손글씨/진한).
- 개발 착수 시 스택은 **Next.js(App Router) 프론트 + NestJS 백엔드 + PostgreSQL + Google OAuth 우선**.
- 변경 후 가능하면 jsdom 등으로 동작을 검증할 것(`outputs/app.test.js`·`editor.test.js`).
- 토큰은 얼마든지 소비해도 됨.
- **병행 세션 2개 운영**: 시작 전 `PLANNING.md`(SSOT) 필독. 충돌 방지 분담 — 앱(`saegim/app.html`) / 에디터(`saegim/editor.html`). WYSIWYG 계약 `comp{bg,dim,textColor,size,weight,align,font}`(에디터)↔`composedInner`/`.cmp-text`(앱)을 바꾸면 양쪽+테스트 동시 수정. 변경마다 PLANNING 진행 로그 1줄.
- 이 폴더는 **git 관리**(기준선 커밋 있음) — 큰 변경 전/후 커밋 권장.
