# 새김 (SAEGIM) — 기획 노트

> 새김(SAEGIM) 독립 프로젝트 · 현재 **기획/디자인 프로토타입 + 실서비스 개발 골격 착수**
> 최종 갱신: 2026-06-29 · 상단(1~14장)=**현재 확정 상태 종합본**, 하단 **진행 로그**=변경 이력(시간순).
> 작업 전 이 문서를 읽고, 변경 시 진행 로그에 1줄 남기기. 진행 로그에는 폐기된 과거 용어가 남아 있을 수 있으므로 현재 기준은 항상 상단 종합본을 우선한다.

---

## 1. 개요 · 한 줄 정의

- **새김(SAEGIM)** = **문장을 카드로 만들어 발견·새김·컬렉션으로 엮는** 모바일 웹 서비스.
- 한 줄: **"한 줄을 카드로 만들어, 발견하고, 마음에 새겨 간직하는 곳."** 깊이 있는 문장 큐레이션 + 가벼운 작가 구독(인스타식).
- 무게중심: **깊은 큐레이션 플랫폼**(문장·독서·카피 애호가 니치). 입구는 가벼운 발견 피드 → '가볍게 들어와 깊게 빠지는' 구조.
- 타겟: 독서·문구 애호가, 카피라이터·기획자, 일상 문구를 기록·공유하려는 SNS 세대.
- **용어 기준**: 사용자 화면에서는 **글/장/둘러보기/글벗**을 쓴다. `Collection`/`COLLECTIONS`는 프로토타입 내부 식별자와 개발 모델 용어로만 유지한다.

## 2. 브랜드 · 정체성

- **브랜드명 '새김'** — ① 좋은 문장을 _마음에 새기다_ ② 글자를 _새겨 남기다_(출판 어원). 동사 '새기다'.
- **하이브리드 톤**: 새김의 감성(마음에 새겨 간직)은 유지하되, 비주얼·공유는 트렌디(인스타 감성카드).

## 3. 비주얼 시스템

- **원칙**: 색은 **'글 카드'(그라데이션)에만**, UI 크롬은 무채색. 외곽선 최소화·버튼은 심리스(채움/보더 지양, 주 액션만 톤 채움).
- **컬러 토큰**: 배경 뉴트럴 미스트 `#F6F5F6` · 잉크(먹보라/차콜) `#38323F` · **포인트=잉크 차콜 `#353039` 모노톤** · 보조 무채색. (분홍 제거 — 첫 적용이 과하게 분홍→모노톤 정정.)
- **카드 배경 프리셋**: 새벽 · 노을 · 안개 · 살구 · 라벤더 · 밤 그라데이션 + 단색(밝은/어두운) + 패턴 + 이미지. **에디터 기본=안개(중성)**. 이미지 배경은 딤(어둡게) 레이어로 가독 확보.
- **폰트(카드 문장 5종 · 에디터에서 선택)**: 고딕(Pretendard) · 명조(Gowun Batang) · 둥근(Gowun Dodum) · 손글씨(Nanum Pen Script) · 진한(Black Han Sans). 워드마크=Gowun Dodum(잉크 밑줄), UI=Pretendard. **굵기 4단**(가늘게 300·보통 400·굵게 700·두껍게 800). 클래스 키 `f-round/f-pen/f-black`(앱·에디터 공유).

## 4. 핵심 개념 모델

- **카드(장)** = 원자 단위(한 종류, **카드 내부 스크롤 없음** — 작성 크기로 표시, 넘치면 카드 가장자리에서 클립). **글** = 발행 단위(게시물)의 UI 명칭 = 카드(장)의 순서 묶음(1~N장).
- **카드(장) ↔ 글 = 1:N**. 1장도, N장도 한 '글'. UI 용어는 **글/장**, 내부 코드 식별자(COLLECTIONS 등)는 유지.
- **결(테마/무드)은 사용자가 직접 안 고름**(카테고리 강요 지양). 비슷한 문장 묶기·추천·취향은 **의미 임베딩(백엔드)** 담당(프로토타입은 클라 similar). 발행 시 입력은 보조 **태그**(사용자 자기 말, 0~3, 선택)뿐.
- **좋아요 ↔ 새김 분리(둘 다 유지)** — _"보내는 것 vs 담는 것"_:
  - **좋아요(♡) = 공개 공감** — 남에게 보내는 가벼운 반응, 수치 공개, 저자 신호.
  - **새김(서랍에 넣기) = 비공개 간직** — 나만 보는 보관함(서랍)에 담아 다시 봄. **브랜드 핵심 액션.**
- **소셜 = 인스타식 평등 계정**: 모든 계정 동등·상호 구독. **인증=신분 아닌 '상태'**(**단일 '공식' 채움 체크 마크 하나** — 공인 2단계는 폐지). 구독 = **글벗**(버튼 라벨은 '구독/구독중'). 과열 요소(경쟁 지표) 절제.
- **프로필 지표 = 글 수 + 글벗 수**(구독자→'글벗' 순화). **문장 수는 비노출**(글과 혼동 방지).

## 5. 정보구조 — 5탭 + 검색

- 하단 **5탭(라벨 없는 아이콘만)**: 홈(집) · 발견(나침반) · ＋포착(＋ FAB) · 둘러보기(책) · 나(프로필 사진). 상단 우측 **검색** 아이콘. **아이콘은 라인 round 마감 통일, 활성=채움.**
- **홈** = 에디토리얼(서비스 주도). **발견** = 글 피드(몰입 탐색). **＋포착** = 카드 에디터(별도 페이지, 서비스의 심장). **둘러보기** = 글 갤러리(사용자 주도). **나** = 통합 프로필.
- 전체 페이지(설정·프로필 편집)는 하단 탭·FAB 숨김.

## 6. 화면별 상세

### 홈 (에디토리얼)

- 상단 **캐러셀 배너 = 서비스 주도 면**: **글(문장 카드) 1개**(유저가 올린 듯한 글 — 무라벨 콘텐츠 슬라이드, 탭 시 **발견 피드의 그 글로**, 별도 페이지/시트 없음) + **공지·이벤트·광고(B2B)** → 퍼블리싱 페이지(`#view-page`). 점 인디케이터·5초 자동 전환·드래그. 광고는 **'AD'로 투명 표기**. (오늘의 한 문장 컨셉·전용 카드 페이지 폐기. **문장 상세 시트도 전면 폐기** — 모든 문장 탭은 발견 피드=카드 상세로)
- **오늘 닿은 글** 가로 레일(미리보기 카드, 데스크톱 마우스 드래그 스크롤).
- **추천 글벗** 가로 레일: 아바타·닉네임(+공식 마크)·**한줄 소개글**·**글 수·글벗 수**·구독 버튼.
- 제거: 테마칩(테마로 둘러보기), '당신을 위한 문장'(미합의 기능).

### 발견 (글 피드)

- **2D 뷰어**: 세로=글 이동, 가로=장(카드) 이동. 모바일 스와이프 + **PC 마우스 휠/키보드 방향키/드래그**. **카드 내부 스크롤 없음** — 문장은 전부 표시, 넘치면 카드 가장자리에서 클립(작성 시 크기로 맞춤).
- 크롬: **상단 중앙 제목 라벨**(카드 상세 최상단, 현재 문장과 같으면 숨김), **하단 좌측 작가 바**(투명 컴팩트 칩 — 아바타·이름·인증·구독, 콘텐츠 크기·상태 줄 없이 절제), 하단 중앙 **장 점 인디케이터**, 우측 **액션 레일**(새김=**북마크**·좋아요=하트·댓글=말풍선, 투명 아이콘·맨 아래 ⋯). 홈·둘러보기·검색·프로필·서랍 등에서 **카드 상세로 진입한 경우에만 좌상단 뒤로가기**를 표시한다(메인 발견 탭은 제외). (작가 바 상단→하단·제목 상단으로 위계 분리.)
- **⋯(우측 액션 레일 맨 아래) → 하단 정보 패널**: 출처(『책』·저자), 태그, 공유하기 / 링크 복사(준비) / **책으로 보기(준비 중, B2B 진입점)** / 신고(준비). 패널 안에는 'B2B' 배지를 드러내지 않는다. ⋯는 탭바에 가리지 않도록 새김·좋아요·댓글 아래에 배치.

### 둘러보기 (글 갤러리)

- 부제 '엮어 둔 글 모음'. 정렬 **인기/최신**(무드 필터는 MVP 보류 — 백엔드 큐레이션 전제). 메이슨리 카드 그리드.
- 카드 진입 → 발견 피드로 점프.

### 검색

- 계정·글 통합 결과 + 빈 입력 시 최근 검색·인기. 계정 메타=한줄 소개글·글벗 수. **문장은 별도 아이템 아님 — 카드 내용 기반 글 검색(colls 필터가 c.q 카드 문장 포함)에 흡수**, 글 카드 미리보기=둘러보기 shelf-card 동일. 뒤로가기는 검색을 연 화면으로 돌아간다. (무드 찾기·문장 결과 제거.)

### 나 / 프로필 (나 = 타인 동일 레이아웃)

- 상단바: 아이콘만(나=옵션, 타인=뒤로). **헤더 닉네임 중복 제거.**
- 헤더: 아바타(내 사진 반영) · 이름+**공식 마크** · **한줄 소개글** · **글 N개 · 글벗 M** · 우측 **컴팩트 액션 버튼**(나=프로필 편집 / 타인=구독·구독중).
- 그 아래 **소개글(인용)** — 한줄 소개글과 다를 때만(따옴표 아이콘 제거).
- **글 그리드 = 둘러보기 카드(shelf-card) 동일 컴포넌트**(메이슨리 2단 · 문장+제목 · 우측 상단 'N장' 배지). **작성자만 제외**(이미 특정), **내 글은 ♡ 숨김**. 내 글 배경=작성 구성(comp)·적응 텍스트색.
- **설정 전체 페이지**: 계정(프로필 편집) / 활동(내 서랍·구독 목록) / 알림(푸시·새김 소식) / 정보(공지사항·이용약관·개인정보·문의) / 로그아웃. 미구현 항목(구독 목록·알림·약관·개인정보·문의)은 **'준비 중'** 라벨, **계정 정보·데이터 초기화 행은 제거**. (내 서랍은 설정에 잔존, 프로필에선 제거.)
- **프로필 편집 전체 페이지**: 프로필 사진(업로드·미리보기) · 닉네임 · 한줄 소개글 · 프로필 소개글(글자수 카운터) → `state.profile` 저장·반영.

### 서랍 (새김 보관함)

- **비공개 보관함**('내가 간직한 글', 부제=간직한 글 수 — '나만 봐요' 라벨은 정리됨, 비공개 성격은 유지). 미리보기 카드 메이슨리 그리드(둘러보기·프로필과 동일 shelf-card)로 한 번에 봄 — 문장+**제목(글 제목, 출처 아님)**+작성자(**닉네임**), 내 글은 **작성 배경(comp)**. 검색·정렬(최신/작가). 새김 시 여기로 모임. (구 티켓·뒤적이기 UI 폐기.) 진입은 설정의 '내 서랍'.

### 글 상세 댓글

- 발견 액션바 **댓글 버튼+개수** → 댓글 시트(목록 + 작성칸). 프로토타입은 `state.comments` 키별 저장 + 샘플 시드.

### 소식 페이지 (공지·이벤트·광고/B2B)

- 배너·설정에서 진입하는 **퍼블리싱 전체 페이지(`#view-page`)**: 헤더(← 뒤로 + 타입) + 본문(라벨·제목·날짜·문단·선택 CTA). FULLPAGE(탭/FAB 숨김), 뒤로는 진입처(홈/설정)로.
- 종류: **공지**(텍스트) · **이벤트**(CTA=발견으로 이동 등) · **광고/B2B**(CTA=제휴 문의·준비 중). 프로토타입은 정적 `PAGES`(향후 운영자 CMS로).
- 설정 > 정보 > **공지사항** = 공지 목록 → 항목 탭으로 페이지 열람(아카이브). 배너는 최신 노출, 목록은 보관.

### 진입 게이트(인증)

- 미로그인 진입 시 **로그인/회원가입**(이메일·비번, Google 우선 OAuth 버튼) + 게스트('로그인 없이 둘러보기'). 프로토타입은 `state.loggedIn` 플래그. 카카오·네이버는 provider 추상화 뒤 후순위로 추가한다.

## 7. 카드 에디터 (＋포착) — 작성 = 조회 동일(WYSIWYG)

- **풀블리드 캔버스**: 카드가 화면을 꽉 채워 조회(발견/뷰어)와 동일 뷰. 발행 시 구성(comp)을 통째로 전달해 어디서나 같게 렌더.
- **문구 배치 = 인스타그램 스토리식**: 본문 **드래그=위치 이동**, **모서리 그립 드래그=글자 크기 유연 스케일**(박스가 콘텐츠 자동 너비라 좌우·상하 함께 커짐), **탭=편집**. 출처도 드래그 이동.
- **인라인 편집(탭 시)**: 입력창 세로 중앙(짧으면 가운데서 시작·오토그로우). 하단 칩+**서브메뉴(팝오버)** — 폰트(5종, 각 폰트 미리보기)·정렬(왼/가운데/오른)·굵기(4단)·크기(슬라이더+프리셋)·**글자색(프리셋+직접 색상 픽커)**.
- **우측 상단 도구(바텀시트) 4종**: 제목 · 배경 · 출처 · 태그. (꾸밈/스티커는 MVP에서 제거.)
  - **배경**: **직접 색상 픽커** + 바로 아래 **직접 이미지(업로드 → `center/cover`로 카드에 꽉 차게, 딤 기본 0.34)** + 프리셋(밝은/어두운 단색·그라데이션·패턴·이미지 프리셋) + **딤(어둡게) 슬라이더**.
  - **출처**: **저자 · 책**(연도 필드 제거). 책 연결 = **준비 중**(작가·출판사 도서 연결, 'B2B' 문구 미표기).
  - **제목**: 비우면 첫 문장이 제목. 입력칸 **placeholder에 첫 문장을 자동 미리보기**. 입력 시 카드 상단 제목 라벨 표시.
  - **태그**: 0~3개 **조용한 키워드**(검색·추천용). **카드 얼굴엔 안 보이고** 앱 ⋯ 상세 패널에만 노출, **# 없는 소프트 펜**(시끄러운 해시태그 지양).
- **발행**: 미리보기 모달 없이 **발행 확인 팝업** 후 핸드오프 큐 적재 → 앱이 수신해 발견·둘러보기·프로필에 반영. **카드 삭제 시도 확인 팝업**(되돌릴 수 없음 안내).
- 톤: 입력칸·링크 보더 완화(무채색 채움+포커스 은은한 링), 버튼 심리스 — 상세보기와 통일.

## 8. 소셜 · 계정 · 인증

- 모든 계정 동등(일반·작가·출판사 동일 모델), 누구나 상호 구독(**글벗**). 프로필은 나=타인 동일 화면(나만 옵션 메뉴).
- **인증 = 상태**: **단일 '공식' 채움 마크 하나**(닉네임 옆, even-odd 체크로 어떤 배경에서도 보임).
- 표시: 닉네임 옆 마크 + 보조 줄 **한줄 소개글**('공식 계정' 마크).

## 9. 데이터 모델

### 개념(백엔드 방향)

- **카드(장)**: 본문 · 구성(comp: 배경·딤·글자색·크기·**굵기(weight)**·정렬·**폰트(고딕/명조/둥근/손글씨/진한)**·위치) · 출처 · 소속 글 id · 순서 · 임베딩. (박스 너비=콘텐츠 자동, 내부 스크롤 없음.)
- **글(Collection)**: 제목 · 작성자 · 카드 목록(순서) · 커버 · 공개여부 · 생성유형(창작/큐레이션) — 카드와 **1:N**.
- **계정**: 프로필(사진·닉네임·한줄 소개글·소개글) · 상호 구독 그래프 · 서랍(새김) · 발행 글 · 인증 상태.
- **출처**: 유형(책/웹/직접/출판사) · 저자 · 책 · URL.
- 임베딩 → 취향 피드 · 유사 추천 · 자동 묶음. 관계: 발행=글(≥1장) · 발견=글/장 · 서랍=새김(비공개) · 좋아요=공개 · 구독=계정↔계정.

### 프로토타입(현재 구현, localStorage·단일 HTML)

- 키: `saegim_v1`(앱) · `saegim_editor_v2`(에디터) · `saegim_pub_queue`(발행 핸드오프).
- 앱 `state`: carved(새김) · liked(좋아요) · subs(글벗) · published(발행 글) · comments · profile{name,photo,bio,tagline} · loggedIn · taste(휴면).
- 에디터→앱: 발행 시 글(제목·장들·comp·테마·태그)을 큐에 적재 → 앱이 진입/복귀 시 흡수.

## 11. 기술 · 플랫폼 · 작업 규칙

- **출시 형태**: 웹(모바일 뷰) + 웹앱(WebView 래핑). 단일 컬럼·세로 스크롤·하단 5탭 + 플로팅 포착. 데스크톱도 모바일 뷰 + 마우스/키보드 보강.
- **데모 원칙**: 의존성 없는 **단일 HTML**(바닐라 JS, C 모노톤 토큰). CDN 폰트 = Pretendard · Gowun Batang · Gowun Dodum · Nanum Pen Script · Black Han Sans. 변경 후 jsdom으로 동작 검증.
- **실서비스 골격**: `apps/web`(Next.js App Router) · `apps/api`(NestJS) · `packages/domain`(공유 타입) · `docker-compose.yml`(로컬 PostgreSQL). 프로토타입은 동작·디자인 레퍼런스로 유지하고 그대로 포팅하지 않는다.
- **저장소**: 현재 `saegim` 폴더 = **git 관리**(기존 ilit 레포와 섞였던 것 분리 완료, '정리 전 기준선' 커밋 존재 → 삭제·실험도 복구 가능).
- **현재 검증**: 변경마다 **jsdom 스모크**(앱·에디터 로드 + 핵심 동작). ※ 테스트는 **세션별 임시(outputs, git 미포함·휘발)** — 개발 착수 시 repo에 정식 포함 필요(§14.8).
- **병행 작업 시 규칙**: PLANNING.md가 SSOT. 충돌 최소화 권장 분담 — 한 세션은 **앱(`saegim/app.html`)**, 다른 세션은 **에디터(`saegim/editor.html`)**. 공유 계약(WYSIWYG): 에디터 `comp{bg,dim,textColor,size,weight,align,font}` ↔ 앱 `composedInner`/`.cmp-text`(폰트 클래스 `f-round/f-pen/f-black`·콘텐츠 자동 너비). **이 계약을 바꾸면 양쪽 파일+테스트 동시 수정**. 각 변경은 진행 로그 1줄(+가능하면 커밋).

## 14. 개발 착수 가이드 (프로토타입 → 실서비스)

> 산출물은 **단일 HTML 프로토타입**(`saegim/app.html` 통합 SPA · `saegim/editor.html` 에디터, 바닐라 JS·localStorage). 실개발 시 아래를 기준으로 전환. design-system.html은 디자인 토큰·화면 데모.
> **개발 세션 인수인계는 `HANDOFF.md`**(빌드 요구사항·계약만, 기획 히스토리 제외 — 편향 방지).

- **기술 스택(확정, 2026-06-29)**: 프론트 **Next.js(App Router)** · 백엔드 **NestJS** · DB **PostgreSQL** · 인증 **Google OAuth 우선**(심사 불필요, provider 추상화한 범용 OAuth 모듈로 설계 → 카카오·네이버 이후 추가). 프로토타입(바닐라 단일 HTML)은 **동작·디자인 레퍼런스**일 뿐, 그대로 포팅하지 말고 Next.js/NestJS 구조로 새로 구현.
- **현재 코드 골격**: `apps/web` · `apps/api` · `packages/domain`으로 시작. `packages/domain`의 카드/글/계정 타입을 프론트 공통 계약으로 사용하고, `apps/api/prisma/schema.prisma`를 PostgreSQL 모델 계약으로 둔다. 웹은 로그인 게이트(로그인/회원가입/Google/게스트) 뒤에서 `NEXT_PUBLIC_API_BASE_URL`로 API를 읽고, 실패 시 샘플 데이터로 유지한다. 포착 탭은 한 장짜리 글을 `POST /posts`로 발행해 발견 피드에 즉시 반영한다. 발견 화면은 현재 글/장 위치 상태로 ↑/↓ 글 이동과 ←/→ 장 이동을 처리하고, 홈·둘러보기·서랍·검색에서 연 글을 발견 화면의 현재 글로 고정한다. 발견 화면의 좋아요/새김 버튼은 API 응답으로 `viewerState`를 갱신한다. 발견 댓글 버튼은 댓글 시트를 열고 `GET/POST /posts/:postId/comments`로 목록·작성 흐름을 처리한다. 나 탭의 `내 서랍`은 `GET /drawer`로 현재 계정이 새김한 글 목록을 읽어 카드 그리드로 표시한다. 상단 검색은 `GET /search?q=`로 계정·글 통합 결과를 보여주고, 글 결과는 발견 화면으로, 계정 결과는 동일 프로필 화면으로 연다. 홈 추천 글벗과 발견 작성자 바의 구독 버튼은 `POST/DELETE /accounts/:accountId/follow` 응답으로 `viewerState.subscribed`를 갱신한다. 홈 추천 글벗·발견 작성자·검색 계정은 타인 프로필로 진입하고, 프로필 진입 시 `GET /accounts/:accountId`로 계정 상세와 공개 글 목록을 보강해 해당 계정 글 그리드와 구독/내 프로필 액션을 표시한다. 나 탭은 `GET /accounts/me` 현재 계정 프로필을 반영하고 `PATCH /accounts/me`로 프로필 편집을 저장한다. API는 Prisma 저장소를 통해 PostgreSQL에 발행·좋아요·새김·구독·댓글을 저장한다. 현재 사용자 컨텍스트는 `DEV_ACCOUNT_ID` 개발 계정 경계로 분리해 두었고, 개발 환경에서는 `x-saegim-account-id` 헤더로 계정 컨텍스트를 임시 전환할 수 있다. 웹은 `NEXT_PUBLIC_DEV_ACCOUNT_ID`가 있으면 이 헤더를 자동으로 보낸다. Google OAuth 세션 도입 시 이 경계를 실제 로그인 계정으로 교체한다.

### 14.1 화면 라우팅 (app.html · 해시 딥링크)

- 탭: `#home` · `#discover`(발견) · `#shelf`(둘러보기) · `#me`(나) / 보조: `#search` · `#drawer`(서랍) · `view-profile`(타인) · `view-settings` · `view-editprofile` · `view-page`(소식).
- `switchView(v)`가 해시 동기화. **FULLPAGE**(settings·editprofile·page)는 하단 탭바·＋FAB 숨김. **＋포착**은 `editor.html` 별도 페이지(새 탭).

### 14.2 상태 · 저장 (localStorage → 실서비스 API/DB)

- `saegim_v1`(앱 `state`): `carved[]`(새김) · `liked{}` · `subs[]`(글벗) · `published[]`(발행 글) · `comments{}` · `profile{name,photo,bio,tagline}` · `loggedIn` · `taste`(휴면).
- `saegim_editor_v2`(에디터 `S`): `pages[]`(각 장 comp) · `current` · `collTitle` · `tags`.
- `saegim_pub_queue`: 에디터→앱 발행 핸드오프.
- 실서비스: 계정 DB · 발행 API · 관계(구독/좋아요/새김) 테이블 · 검색 인덱스로 이관.

### 14.3 발행 흐름

- 프로토타입: 에디터 `enqueuePublication()` → `saegim_pub_queue` 적재 → 앱 `ingestPublications()`/`drainPubQueue()` → `state.published` + `rebuildData()`(DATA=SEED+published) → 발견·둘러보기·프로필·서랍 반영.
- 실서비스 골격: 웹 포착 탭 → `POST /posts` → API가 글/장 bundle 생성·PostgreSQL 저장 → 웹 피드 상태에 prepend → 발견 탭 반영. 발행 글과 좋아요·새김 상태는 서버 재시작 후에도 유지된다.

### 14.4 WYSIWYG comp 계약 (필수 유지)

- `comp{bg,dim,textColor,size,weight,align,font(,textPos)}`. 에디터 작성 = 앱 `composedInner`/`composedBg`/`composedTextColor`로 동일 렌더. 폰트 클래스 `f-round/f-pen/f-black`, 박스=콘텐츠 자동 너비·**카드 내부 스크롤 없음**. **이 계약 변경 시 app·editor·테스트 동시 수정.**

### 14.5 데이터 시드 (현재 정적 → DB/CMS)

- `SEED`(시드 문장) · `COLLECTIONS`(큐레이션 글) · `PUBS`/`AUTHORS`(추천 계정) · `PAGES`(소식, →운영자 CMS) · `VERIFY`(인증=공식 단일).

### 14.6 백엔드가 담당할 것 (프로토타입은 임시 대체)

- **의미 임베딩**(취향 피드·유사 추천·자동 묶음) — 프로토타입의 클라 `similar()`는 폐기, '결' 사용자 선택 없음.
- **인증/OAuth**(Google 우선, 카카오·네이버 후순위) — 현재 프로토타입은 `loggedIn` 플래그.
- **발행 저장·관계 그래프·검색 인덱스** — 현재 클라 필터/큐.
- 현재 API 계약: `GET /auth/google` · `GET /auth/google/callback` · `GET /feed` · `GET /shelf` · `GET /posts/:postId` · `GET/PATCH /accounts/me` · `GET /accounts/recommended` · `POST /posts` · `POST/DELETE /posts/:postId/like` · `POST/DELETE /posts/:postId/carve`(PostgreSQL 저장, 시드는 서버 시작 시 idempotent 보강).
- 현재 사용자 계정 경계: `apps/api/src/auth`의 개발용 현재 계정 서비스가 `DEV_ACCOUNT_ID`(기본 `acct-me`)를 반환한다. 개발 환경에서는 `x-saegim-account-id` 헤더를 계정 힌트로 받아 좋아요/새김/추천 제외 상태를 계정별로 검증한다. OAuth/session 연결 시 콘텐츠 저장소가 이 경계를 통해 실제 계정을 받도록 교체한다.
- 웹 개발 계정 힌트: `NEXT_PUBLIC_DEV_ACCOUNT_ID`를 설정하면 웹 API 클라이언트가 개발 환경에서만 `x-saegim-account-id`를 자동 첨부한다.
- 웹 로그인 게이트: 실서비스 골격에서는 `saegim_web_entry_state` localStorage로 로그인/게스트 입장 상태만 임시 저장하고, 나 탭 로그아웃으로 이 값을 지운다. 실제 Google OAuth 세션이 붙으면 이 임시 플래그를 제거한다.
